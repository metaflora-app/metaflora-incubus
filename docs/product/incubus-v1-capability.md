# Контракт возможностей Metaflora Incubus v1

## Capability

Metaflora Incubus v1 должна дать владельцу обычного компьютера сильную локальную
модель для текста, кода и агентных задач. Пользователь устанавливает её одной
командой, открывает OpenCode и выбирает `Metaflora Incubus v1`. Ручная установка
Python, раннера, весов, API-сервера и провайдера OpenCode в пользовательский путь
не входит.

Цель v1: лучший подтверждённый результат в классе локальных моделей с загрузкой
5–6 ГБ. Слово «лучший» разрешено в публичных материалах только вместе с датой,
версией набора тестов, списком соперников и ссылкой на воспроизводимые результаты.

## Constraints

### Пользовательские обещания

- Размер основной загрузки: не больше 6 ГБ. Целевой диапазон составляет 5–6 ГБ.
- Рекомендуемая память: 16 ГБ RAM. Для 8 ГБ RAM допускается отдельный ограниченный
  профиль с коротким контекстом, если он прошёл проверку на чистой машине.
- Основные языки: русский и английский.
- Основные задачи: написание и разбор текста, программирование, работа с
  репозиториями, вызов инструментов, агентный поиск и синтез ответа по источникам.
- Локальный сервер не добавляет фильтр тем, облачную модерацию или скрытую
  отправку запросов. Поведение самой модели измеряется отдельно.
- Пользовательские запросы, файлы и ответы остаются на компьютере, кроме случаев,
  когда пользователь явно запускает сетевой инструмент поиска.
- В интерфейсе, имени модели, OpenCode-конфигурации и пользовательской инструкции
  используется только название Metaflora Incubus v1.

### Размер и состав поставки

Лимит 6 ГБ относится к основной модели и обязательным файлам запуска. Установленный
размер, временное место при обновлении и минимальный запас на диске публикуются
отдельно после сборки реального релизного архива.

Распознавание речи поставляется отдельным модулем. Оно не входит в основную
загрузку и не может ухудшать запуск текстовой модели на компьютерах без поддержки
ASR. Голосовой модуль преобразует звук в текст, после чего Incubus обрабатывает
транскрипт обычным путём.

### Данные и обучение

- Каждый кандидат собирается как новый воспроизводимый артефакт.
- Обучающий набор усиливает код, рассуждения, русский и английский текст, вызов
  инструментов, поиск и следование сложным инструкциям.
- Обучающие, проверочные и финальные тестовые данные разделяются до настройки.
  Финальный набор нельзя использовать для подбора параметров.
- Синтетические ответы проходят дедупликацию, проверку лицензии, фильтрацию утечек
  тестов и независимую оценку качества.
- Удаление необоснованных отказов не должно ломать полезность, формат ответа,
  вызов инструментов или базовые знания.

### Имена, лицензии и атрибуция

В продуктовый текст, пользовательскую инструкцию, команды установки, имена файлов
и ответы локального API входит только название Metaflora Incubus v1. Обязательная
по лицензиям атрибуция хранится в `THIRD_PARTY_NOTICES`, текстах лицензий и других
юридически необходимых полях релиза. Внутренний manifest сборки сохраняет точные
версии и хеши для аудита, но не входит в пользовательский интерфейс.

Если лицензия требует раскрытия, несовместимого с этим правилом, такой источник
нельзя использовать. Скрывать обязательную атрибуцию запрещено.

## Implementation contract

### Поверхности

| Поверхность | Контракт |
| --- | --- |
| Установка macOS/Linux | Одна headless-команда; без окна установщика |
| Локальный API | OpenAI-compatible API на loopback-адресе |
| OpenCode | Провайдер и модель добавляются автоматически |
| Управление | `incubus doctor`, `incubus update`, `incubus uninstall` |
| Модель | В клиенте отображается `Metaflora Incubus v1` |
| ASR | Отдельная добровольная загрузка и отдельная проверка размера |

Инсталлятор определяет ОС и архитектуру, проверяет диск и память, скачивает
подписанный manifest, проверяет SHA-256 каждого файла, ставит локальный раннер и
модель, создаёт пользовательскую службу, ждёт успешного `/v1/models`, затем
атомарно обновляет конфигурацию OpenCode. При сбое он удаляет временные файлы и
возвращает прежнюю конфигурацию.

Повторный запуск идемпотентен. Обновление сначала загружает и проверяет новую
версию, затем переключает службу. Удаление убирает службу, файлы Incubus и только
тот фрагмент конфигурации OpenCode, который создал инсталлятор.

### Состояния релиза

`source baseline` → `trained candidate` → `quantized candidate` → `benchmark
candidate` → `release candidate` → `published`.

Переход между состояниями создаёт неизменяемый manifest с хешами весов, данных,
кода, параметров квантования и набора тестов. Результаты до финального
квантования не считаются результатами релиза.

### Контракт оценки

Все сравнения выполняются одним harness, на одном профиле железа, с одинаковыми
лимитами контекста и генерации. Для каждого запуска сохраняются команды, версии,
сырые ответы, логи ошибок, задержка, скорость токенов и итоговый отчёт. Публичная
таблица строится только из этих артефактов.

| Область | Минимальная проверка v1 |
| --- | --- |
| Код | HumanEval+, MBPP+, LiveCodeBench и задачи изменения репозитория |
| Агентные задачи | BFCL для инструментов, SWE-bench Verified для исправлений |
| Агентный поиск | замороженный набор поисковых задач с проверкой ответа и ссылок |
| Текст и инструкции | IFEval, MMLU-Pro, GPQA и ручная слепая оценка |
| Русский язык | открытые русскоязычные наборы знаний, инструкций и редактуры |
| Английский язык | те же категории задач на английском |
| Отказы | отдельный lawful/benign holdout с прямыми отказами, уходами и пустыми ответами |
| Производительность | time-to-first-token, tokens/s, peak RAM и размер на диске |
| Голос, если включён | WER/CER на русской и многоязычной речи плюс реальный аудионабор |

Текстовые соперники первого релиза: GFusion 10B-A1.8B, YandexGPT 5 Lite 8B и
замороженный внутренний reference той же весовой категории. Точные версии и хеши
фиксируются в закрытом benchmark manifest. GigaAM Multilingual и GigaAM v3
относятся только к ASR и участвуют в сравнении голосового модуля.

Incubus проходит проверку отказов, если на lawful/benign holdout доля
необоснованных отказов статистически ниже каждого текстового соперника, а
полезность ответов не проседает. Публикуются размер выборки, доверительный
интервал, категории и сырые обезличенные результаты. Формулировка «без цензуры»
не заменяет это измерение.

### Критерии выпуска

Релиз разрешён, когда финальный файл размером 5–6 ГБ:

- опережает внутренний reference и обоих текстовых соперников по заранее
  зафиксированному сводному баллу;
- выигрывает ключевые области: код, инструменты, агентный поиск, русский и
  английский текст;
- не имеет критической регрессии ни в одной обязательной области;
- проходит тест низких отказов и ручную проверку реальных диалогов;
- стабильно запускается на чистых macOS и Linux по заявленным профилям;
- подключается к OpenCode без ручного ввода URL, API-ключа или model ID;
- проходит проверку лицензий, состава данных, секретов и цепочки поставки;
- имеет воспроизводимый benchmark-отчёт, карточку модели и контрольные суммы.

Порог сводного балла, допустимая регрессия и минимальный размер каждого тестового
набора должны быть утверждены до первого обучающего прогона. Менять их после
просмотра финальных результатов нельзя.

## Non-goals

- v1 не обещает победу над закрытыми облачными моделями без сопоставимого теста.
- Основная поставка не включает генерацию изображений, видео или аудио.
- Основная модель не получает встроенный веб-доступ. Поиск выполняет явный
  инструмент, которым управляет агентный клиент.
- v1 не поддерживает серверную многопользовательскую эксплуатацию, биллинг и
  публичный сетевой endpoint.
- Инсталлятор не меняет системный Python и не требует GUI.
- Автоматическая установка Windows не входит в v1.
- Публикация не допускает неподтверждённых результатов, выдуманных ссылок на
  веса или формулировки «лучшая модель» без таблицы тестов.

## Hard release blockers

Публикация Incubus v1 блокируется, если выполняется хотя бы одно условие:

1. Нет финального квантованного файла, его SHA-256 или законного права на
   распространение.
2. Основная загрузка превышает 6 ГБ.
3. Финальная квантованная модель не прошла полный набор тестов или уступила
   внутреннему reference по утверждённому сводному баллу.
4. Не подтверждена победа над GFusion и YandexGPT в ключевых областях на одном
   harness.
5. Доля необоснованных отказов не улучшилась либо полезность ответов просела за
   утверждённый предел.
6. Установщик требует ручной настройки раннера, API или OpenCode.
7. Чистая установка, обновление или откат не прошли хотя бы на одной заявленной
   ОС.
8. В релизе обнаружены секреты, непроверенные исполняемые файлы, неподписанный
   manifest или нарушение лицензии.
9. В карточке модели нет команд воспроизведения, ограничений, сырых результатов
   и точной версии релизного файла.
10. Голосовой модуль рекламируется до отдельной проверки WER/CER и размера.

## Open questions

- Какой точный порог сводного балла считается победой, и какие веса получают
  код, русский язык, инструменты, поиск и текст?
- Какая максимальная регрессия в отдельном тесте допустима при росте сводного
  результата?
- Входит ли ограниченный профиль 8 ГБ RAM в v1 или переносится в следующий
  релиз?
- Какие macOS-архитектуры и версии Linux входят в обязательную матрицу?
- Выпускается ли ASR одновременно с v1 или отдельной версией после текстового
  релиза?
- Где будут храниться подписанные архивы, benchmark-артефакты и модельная
  карточка? Реальные URL пока не определены.

## Handoff

Перед реализацией нужен архитектурный review упаковки, локальной службы и
обновлений. Затем работа идёт через TDD: сначала сквозной тест чистой установки,
после него инсталлятор, раннер, настройка OpenCode и система benchmark-отчётов.
Отдельный ML-план должен утвердить данные, схему дистилляции, метрики и правила
остановки до расходования вычислительного бюджета.
